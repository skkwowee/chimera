\begin{abstract}
Vision-language models (VLMs) struggle with domain-specific visual grounding tasks in competitive gaming environments, where rapid and accurate scene understanding is critical for strategic decision-making. We propose a two-phase training paradigm: first, supervised fine-tuning on screenshot--game state pairs teaches the model accurate visual grounding (reading the HUD); then, Group Relative Policy Optimization with demo-derived rewards teaches strategic reasoning scored against professional play and round outcomes. We evaluate our approach on Counter-Strike 2 (CS2) screenshot analysis tasks, demonstrating that separating perception and reasoning into distinct training phases outperforms single-phase approaches.
\end{abstract}

\section{Introduction}
\label{sec:intro}

Competitive esports titles like Counter-Strike 2 (CS2) present a rich testbed for vision-language models: the game state is partially observable through a heads-up display (HUD), and expert play generates thousands of hours of professionally annotated replay data (demos). An AI copilot that reads the screen and provides strategic advice must first \emph{perceive} the game state accurately---health, armor, weapons, player counts, bomb status---before it can \emph{reason} about what to do next.

We find that even frontier VLMs struggle with this seemingly simple perception task. Claude Opus 4.6 achieves only 50\% per-field accuracy on CS2 HUD reading, while base Qwen3.5-35B-A3B scores 50\% (with significant format compliance issues). The CS2 HUD uses stylized fonts, dynamic overlays, and context-dependent iconography that fall outside general pretraining distributions.

Our key hypothesis is that perception and reasoning benefit from separate training phases. Phase~1 (SFT) grounds the model in accurate HUD reading using screenshot--game state pairs extracted from demo files. Phase~2 (GRPO) teaches strategic reasoning using rewards derived from professional play outcomes. This separation lets each phase use the most appropriate training signal and avoids the conflation of perceptual errors with strategic mistakes during reward computation.

Our contributions are: (1)~a demo-synchronized data pipeline that pairs CS2 screenshots with engine-accurate ground truth across 4 maps (4{,}353 screenshots, 5{,}309 labels); (2)~a parameter-efficient SFT recipe that achieves 67.3\% per-field accuracy, a +17pp improvement over both Opus~4.6 and base Qwen baselines; and (3)~a two-phase training framework with RECALL, a retrieval-based advantage estimator for GRPO rewards.

\subsection{Phase 1: Visual Grounding via SFT}
\label{sec:sft}

Phase~1 teaches the model to read the CS2 HUD accurately. We construct training pairs $(x_i, y_i)$ where $x_i$ is a screenshot captured during demo playback and $y_i$ is the corresponding structured game state extracted from the demo file at the matching tick. Ground truth includes 11 fields: \texttt{map\_name}, \texttt{round\_phase}, \texttt{player\_side}, \texttt{player\_health}, \texttt{player\_armor}, \texttt{player\_money}, \texttt{weapon\_primary}, \texttt{weapon\_secondary}, \texttt{alive\_teammates}, \texttt{alive\_enemies}, and \texttt{bomb\_status}.

Unlike approaches that rely on model-generated labels, our demo-synchronized pipeline provides engine-accurate ground truth without human annotation. Each screenshot is paired with the exact game state from the demo's tick data via temporal alignment between VOD frames and replay ticks.

We fine-tune Qwen3.5-35B-A3B (MoE, 35B total / 3B active parameters) using LoRA~\cite{hu2022lora} with rank $r{=}4$, $\alpha{=}8$, and dropout $0.1$. Both vision encoder and language model layers are trainable during SFT, enabling the model to adapt its visual features to the CS2 HUD's specific iconography. The model is trained to output valid JSON matching the ground truth schema, with a learning rate of $1 \times 10^{-5}$ over 1 epoch (304 steps) on 3{,}322 training samples.

\section{Experiments}
\label{sec:experiments}

We evaluate on the HUD perception task: given a CS2 screenshot, output a structured JSON with 11 game state fields. Accuracy is measured per-field (exact match) over a held-out test set of 10 samples (seed=42) spanning all 4 maps.

\subsection{Setup}
\label{sec:setup}

\paragraph{Data.} We parse 4 professional demo files (Furia vs Vitality, IEM Katowice 2025) across Mirage, Inferno, Nuke, and Overpass. Demo playback with automated screenshot capture yields 4{,}353 screenshots. Temporal alignment with demo tick data produces 5{,}309 labeled game state instances. After deduplication and quality filtering, 3{,}322 samples are used for SFT training.

\paragraph{Model.} Qwen3.5-35B-A3B~\cite{yang2024qwen2} is a Mixture-of-Experts VLM with 35B total parameters and 3B active per token. We train in bf16 on 2$\times$ NVIDIA H100 SXM GPUs.

\paragraph{SFT hyperparameters.} LoRA rank $r{=}4$, $\alpha{=}8$, dropout $0.1$, learning rate $1 \times 10^{-5}$ (cosine schedule), batch size 8 (gradient accumulation 4), 1 epoch (304 steps), 1h40m wall time. Both vision encoder and language layers are trainable. An initial run with $r{=}32$ over 3 epochs severely overfit (train loss 0.033 / val loss 17.5), motivating the aggressive regularization.

\paragraph{Evaluation.} We report per-field exact-match accuracy over 11 HUD fields. Each field is compared independently; overall accuracy is the mean across all evaluated fields.

\subsection{Baselines}
\label{sec:baselines}

We compare against three zero-shot baselines on the same 10-sample test set:
\begin{itemize}
    \item \textbf{Claude Opus 4.6}: Anthropic's frontier VLM, accessed via API.
    \item \textbf{Claude Sonnet 4.6}: Mid-tier Anthropic VLM.
    \item \textbf{Qwen3.5-35B-A3B (base)}: The unfine-tuned base model, using \texttt{enable\_thinking=False} and temperature 0.6 to produce valid JSON (default thinking mode outputs prose, yielding 0\% format compliance).
\end{itemize}

\paragraph{Key observations.}
\begin{itemize}
    \item \textbf{Numerical fields are solved}: health, armor, money, and round phase all reach 100\%, suggesting the model learns to read CS2's numeric HUD elements reliably.
    \item \textbf{Spatial reasoning remains hard}: alive player counts require interpreting small avatar icons; map identification requires recognizing environmental cues. These fields plateau at 40--50\%.
    \item \textbf{Format compliance}: SFT produces valid JSON on 10/10 samples, compared to 0/10 for base Qwen in default mode.
    \item \textbf{Overfitting is the primary risk}: with only 3{,}322 samples and 41.5M LoRA parameters, the params-to-tokens ratio must be carefully managed. Our $r{=}4$ configuration yields a 5:1 ratio, near the empirical threshold.
\end{itemize}

\section{Analysis}
\label{sec:analysis}

\paragraph{What SFT learns.} The stark accuracy gap between numerical fields (100\%) and spatial/contextual fields (40--50\%) reveals what fine-tuning on 3{,}322 samples can and cannot teach. Numeric HUD elements (health bar, armor indicator, money counter) occupy fixed screen positions with consistent fonts---a small amount of data suffices to learn these mappings. In contrast, alive player counts require interpreting variable-length rows of small avatars, and bomb status depends on game phase context (planted vs.\ carried vs.\ dropped).

\paragraph{The overfitting boundary.} Our experience with $r{=}32$ (catastrophic overfitting at 257:1 params-to-tokens ratio) versus $r{=}4$ (healthy training at 5:1) establishes a practical scaling law for LoRA on small VLM datasets. With vision layers trainable, the effective parameter count includes vision LoRA adapters, making rank reduction doubly important.
